# IEXA Agent Plan

- 第一优先级：让 app 内的 agent 更像真实开发助手，而不是会输出代码文本的聊天框。
- 当前重点方向：
  - 收紧“自动项目生成”触发条件，避免普通聊天被误判成项目
  - 强化 `latest` 工作区连续性，让后续对话能接着已有项目、已有验证结果继续改
  - 让流式输出更丝滑，同时保持流式阶段和最终阶段的排版一致
  - 让终端能力更接近真实终端，保留快捷键条与隐藏输入桥
- 终端方向：
  - 远端 Linux shell 已接入
  - 后续应补 shell capabilities UI、shell 名称显示和 Local/Remote 双模
- 项目工作流方向：
  - latest 工作区是默认连续开发区
  - 项目写入后应同步远端工作区并自动执行验证命令
  - 后续应继续解决远端旧文件清理问题，避免验证受残留文件污染
- agent 行为方向：
  - 普通问答：正常回答，不进入文件模式
  - 项目生成：完整输出显式文件载荷
  - 继续开发 / 修报错 / 跑测试：优先根据当前工作区和最新验证结果推进
